# 2026-07-08

## 오늘 공부한 것

- 배열 원소 이동, 디버깅, 메모리, 포인터

## 핵심 개념

**배열 원소 이동**
- 오른쪽으로 한 칸씩 이동할 땐 for문에서 `i--` (뒤에서부터 진행)
- 왼쪽으로 한 칸씩 이동할 땐 for문에서 `i++` (앞에서부터 진행)
- 이동 방향과 반대쪽 끝에서 시작해서 이동 방향으로 순회해야 값이 안 사라짐

**디버깅**
- 디버깅 시 확인해야 할 것은 제어 흐름과 변수의 값
- 코드에 버그가 있어도 상황에 따라 정상 동작하는 경우도 있음(운 좋게 문제가 안 드러나는 케이스)

**메모리**
- 메모리는 프로그램 실행에 필요한 정보(값)를 저장하는 공간
- 1byte(8bits) 단위로 물리 주소가 부여되며, 일렬로 연속된 1byte 크기의 방들의 모음
- 주소는 16진수로 표현됨

**변수**
- 변수는 선언될 때 메모리에 공간이 할당되고, 한 번 할당되면 고정됨
- 변수에 할당되는 메모리 주소는 시스템마다 다름
- 변수는 두 가지 의미를 가짐 → 할당된 공간 자체, 그리고 그 공간에 저장된 값

**포인터**
- 포인터는 주소를 나타내는 특수 자료형(양의 정수로 표현되지만 `int`와는 별개의 타입)
- 선언: 변수명 앞에 `*` 사용
- 연결: `&`(주소연산자)로 포인터 변수를 다른 변수에 연결
- 참조: `*`로 포인터에 연결된 변수에 접근
- 선언과 동시에 초기화 예: `int num, *pnum = &num;`
- `*pch = 'B';` → pch가 가리키는 공간에 'B' 대입 (간접접근) / `ch = 'B';` → 직접 접근
- 예시:
```c
  int num = 9, *pnum = &num;
  printf("%p\n", &pnum);   // pnum 자체의 주소
  printf("%p\n", pnum);    // pnum이 가리키는 num의 주소
  printf("%d\n", *pnum);   // pnum이 가리키는 num의 값
```
- 포인터는 선언 후 반드시 어떤 변수에 연결한 뒤 사용해야 함
- NULL 포인터: 주소값 0을 나타내는 특별한 기호, 아무것도 가리키지 않음을 의미. 값이 0이므로 조건문에서 거짓으로 취급됨
- 포인터의 자료형과 연결된 변수의 자료형은 일치해야 함
- 포인터 종류(자료형)에 관계없이, 주소를 저장하기 위해 필요한 공간의 크기는 동일함

**배열과 포인터**
- `ar` : 배열 0번 원소의 주소 / `&ar` : 배열 전체의 주소
- `*ar` : 0번 원소의 값
- `ar+i` : i번째 원소의 주소 / `*(ar+i)` : i번째 원소의 값
- 포인터는 보통 배열 전체가 아니라 배열의 첫 번째 원소 주소에 연결됨
- 배열과 포인터는 둘 다 주소를 의미하므로 동일한 방식으로 사용 가능
- 포인터를 배열 중간 원소에 연결할 수도 있음 (예: `int *p = &ar[2];` → `*p == *(ar+2)`)
- 포인터 변수의 증감량은 가리키는 배열 원소의 크기가 아니라, 포인터 자신의 자료형에 의해 결정됨
- 변수에 할당된 주소(포인터 변수 자신의 위치)는 변경 불가능하지만, 변수에 저장된 주소값(포인터가 가리키는 대상)은 변경 가능함

## 복습 포인트

- `&pnum` / `pnum` / `*pnum` 세 가지의 차이 — 각각 포인터 자신의 주소, 포인터가 가리키는 주소, 그 주소에 있는 값을 의미. 헷갈리기 제일 쉬운 부분
- `ar`과 `&ar`은 값(주소)은 같아 보여도 타입이 다름 — `ar`은 `int*`(원소 하나를 가리키는 포인터), `&ar`은 `int(*)[n]`(배열 전체를 가리키는 포인터). 그래서 `ar+1`과 `&ar+1`은 실제로 이동하는 크기가 다름
- 포인터 증감량은 "포인터 자신의 자료형"이 기준 — 포인터는 가리키는 변수와 타입이 일치해야 하므로, 같은 타입일 때는 증감량이 배열 원소 크기와 같아 보이는 게 당연함. 진짜 차이는 타입이 다른 포인터로 같은 배열을 가리킬 때 드러남 `int ar[3]={1,2,3}; char *pc=(char*)ar;` → `pc++`는 1바이트만 이동 (int 원소 크기 4바이트와 다름)
- 변수에 할당된 주소(변수 자신의 위치)는 변경 불가능하지만, 변수에 저장된 값(포인터가 가리키는 대상)은 변경 가능함
