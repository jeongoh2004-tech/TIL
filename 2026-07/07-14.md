## 오늘 공부한 것

- 포인터, 문자열

## 핵심 개념

**값에 의한 호출 vs 주소에 의한 호출**
- 주소에 정수를 더하거나 뺄 수 있고, 주소끼리 비교도 가능함 → 이를 이용해 배열을 훑어볼 수 있음
- 값(정수, 문자)을 인자로 넘기면(값에 의한 호출), 함수 안에서 그 값을 바꿔도 지역변수만 바뀌는 거라 원본에는 영향 없음. 함수가 끝나면 그 지역변수는 사라지기 때문
- 반면 변수의 주소(`&a`)를 넘기면(주소에 의한 호출), 함수 안에서 포인터로 간접 참조(`*p = 10;`)해서 호출한 쪽의 원본 변수 값을 실제로 바꿀 수 있음
- 두 호출 방식 모두 근본적으로는 "값을 복사해서 넘긴다"는 동작 자체는 동일함 — 다만 넘기는 값이 데이터냐 주소냐의 차이

**함수와 배열 인자**
- 함수의 형식 인자 `int x[]`는 사실 포인터임(배열이 아님) → `int x[]`와 `int *x`는 문법적으로 완전히 동일
- 보통 함수 본체에서 배열처럼 다루고 싶을 때는 배열 형태(`int x[]`)로 씀
- 배열을 인자로 넘길 때도 배열의 주소가 전달되므로, 호출된 함수에서 배열 값을 변경하면 원본도 바뀜
- scanf에서 `&`를 붙이는 이유는 입력받은 값을 원본 변수에 저장하려면 주소가 필요하기 때문. printf는 값만 넘기면 되므로 `&` 불필요
- 함수 반환값으로도 주소(포인터)를 사용할 수 있음 → 함수 이름 앞에 `*`를 붙여서 표시

**포인터 배열 vs 배열 포인터**
- 포인터 배열: `int *p[3];` → 배열이며, 원소가 포인터, 포인터 3개를 담고 있음. 각각 다른 대상을 가리킬 수 있음
- 배열 포인터: `int (*p)[3];` → 포인터이며, 배열 하나를 가리킴. 포인터는 1개뿐
- `int` 포인터 변수의 자료형은 `int*`, 그 포인터 변수의 주소의 자료형은 `int**`
- 다중 포인터란 포인터 변수를 가리키는 포인터를 의미함

**문자열**
- 문자열은 연속적으로 나열된 문자들의 묶음, `""`로 감싸서 표현, 서식문자는 `%s`
- 문자열의 끝은 배열의 크기가 아니라 널 문자(`'\0'`)에 의해 결정됨 → 문자 배열에 문자열을 저장하려면 배열 크기가 문자열 길이보다 하나 더 커야 함(널 문자 자리 확보)
- `printf`의 `%s`는 전달된 주소부터 널 문자 직전까지 출력
- `scanf`의 `%s`는 개행/공백/탭 문자 직전까지를 하나의 문자열로 인식
- 변수 영역에 저장된 문자열은 내용 변경 가능, 상수 영역에 저장된 문자열은 변경 불가능 → 상수 문자열을 바꾸고 싶으면 내용을 수정하는 게 아니라 포인터가 다른 문자열을 가리키게 해야 함
- 문자열 포인터 배열은 여러 문자열 자체를 저장하는 게 아니라, 각 문자열의 시작 주소들을 저장하는 배열

**문자열 관련 함수**
- `strlen(s)`: 문자열 s의 길이 반환
- `strcpy(s1, s2)`: s1에 s2를 복사
- `strcat(s1, s2)`: s1의 끝에 s2를 접합
- `strcmp(s1, s2)`: 사전 순 비교(s1<s2면 음수, 같으면 0, s1>s2면 양수 반환)
- str 함수 뒤에 n을 붙이면 길이를 지정할 수 있음(예: `strncpy`)
- `atoi(s)`, `atof(s)`, `atol(s)`: 문자열을 각각 int, double, long으로 변환
- `puts(char *str)`: 문자열 출력 후 마지막에 `'\n'` 자동 출력, 성공 시 음수 아닌 값·실패 시 EOF 반환
- `gets(char *s)`: 엔터 전까지 입력받고 `'\n'`은 버린 뒤 `'\0'` 붙임. `gets_s()`는 gets()의 보안 버전으로 배열 크기를 인자로 전달, `fgets(str, 10, stdin)`도 안전하게 사용 가능
- `putchar(int c)`: 문자 하나 출력
- `getchar(void)`: 문자 하나 입력받아 반환, 실패 시 EOF 반환
