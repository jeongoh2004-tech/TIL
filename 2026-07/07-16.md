## 오늘 공부한 것

- 구조체

## 핵심 개념

**구조체란**
- 의미상 관계가 있는 항목을 그룹으로 묶어 표현한 자료형
- `int`, `char`처럼 변수의 "모양"을 의미한다는 점은 같지만, `int`/`char`는 미리 정해진 기본 자료형인 반면 구조체는 사용자가 용도에 맞게 만들어 쓰는 사용자 정의 자료형
- 구조체를 구성하는 변수들을 "멤버"라고 부름

**구조체 정의와 선언**
```c
struct 구조체자료형이름 {
    멤버자료형 멤버변수;
    멤버자료형 멤버변수;
    ...
};   // 정의만 해서는 메모리에 공간이 할당되지 않음

struct student st1, st2;   // 선언 → 이때 비로소 메모리 공간이 할당됨
```

**구조체 변수 사용**
- 멤버 접근은 구조체 멤버 연산자(`.`) 사용
- 중괄호 안에 멤버 값을 나열하여 초기화 가능
- 구조체 변수의 대입 연산은 각 멤버 변수에 대한 대입 연산을 의미함(멤버별로 값이 복사됨)
- 구조체 크기가 필요하면 `sizeof` 연산자 사용

**구조체 포인터**
- 선언은 일반 포인터와 동일하게 `*` 사용
- `&`(주소 연산자)로 구조체 변수의 시작 주소를 얻음
- 포인터가 가리키는 구조체 멤버에 접근하려면 간접 연산자 `*` 사용 — 단, `*`보다 `.`의 우선순위가 높아서 반드시 괄호 필요: `(*pst).id`
- 구조체 포인터 전용 연산자 `->` 사용 가능: `(*pst).id` == `pst->id`

**구조체와 함수**
- 구조체를 함수 인자로 사용: 실인자의 값이 형식인자에 그대로 대입됨 (실인자 = 함수를 호출하는 쪽의 매개변수, 형식인자 = 함수를 선언하는 쪽의 매개변수)
- 구조체를 함수 반환형으로 사용: 구조체 전체 값이 통째로 호출한 쪽에 전달됨
- 구조체 포인터 변수를 함수 인자로 사용: 실인자의 값(주소)이 형식인자에 대입됨
- 구조체의 주소를 반환하는 함수: 일반 변수의 주소를 반환하는 것과 동일한 방식

**중첩(자기참조) 구조체**
- 자기 자신과 동일한 구조체 자료형은 멤버로 사용할 수 없음(크기가 무한히 커지는 모순이 생기기 때문)
- 자기 자신과 동일한 구조체의 **포인터**는 멤버로 사용 가능(포인터는 크기가 고정되어 있어서 문제없음)

예시:
```c
struct student7 {
    int id;
    char name[8];
    double grade;
    struct student7 *roommate;      // 문제 없음
    struct student7 *friends[10];   // 문제 없음
};

struct student7 st1 = {10, "Tom", 3.2};   // roommate, friends는 명시적으로 초기화 안 했으므로 자동으로 NULL
struct student7 st2 = {11, "Jerry", 4.3}; // 마찬가지로 roommate, friends는 NULL

st1.roommate = &st2;   // st1의 룸메이트는 st2
printf("roommate's name: %s\n", st1.roommate->name);
```

**typedef로 사용자 자료형 이름 재정의**
```c
typedef unsigned char *UCHARPTR;
UCHARPTR p;   // unsigned char *p; 와 동일한 선언
```
- 구조체는 `struct` 키워드 때문에 선언이 길어지는데, `typedef`로 이름을 재정의하면 코드가 간결해짐
